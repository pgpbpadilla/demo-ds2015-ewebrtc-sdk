Namespace: rtc

rtc

This namespace provides the methods necessary to configure the JS client library.

  • You can use the configure method to request environment options from the built-in DHS. Alternatively you can also configure using your own environment options.
  • You can use the associateAccessToken method to associate an access token with a virtual number or account id user id.
  • Use getEWebRTCDomain to obtain the WebRTC domain setup for the library.

<static> configure(options, success, error)

Setup the Enhanced WebRTC API endpoint and Enhanced WebRTC domain.

If options object is provided, configure method will use the options.ewebrtc_domain and [options.api_endpoint] values as the application domain and api endpoint respectively. If no options were passed, the configuration will be requested from the built-in DHS and then applied.

Parameters:
Name Type Argument Description
options Object <optional>

SDK configuration options.

Properties
Name Type Argument Description
ewebrtc_domain String

The Enhanced WebRTC Domain for your application.
api_endpoint String <optional>

The Enhanced WebRTC API URL.
success function <optional>
error function <optional>
Examples
// Configure using the Node DHS
var success = function () {...};
ATT.rtc.configure(success);
// Configure using the Node DHS
var success = function () {...};
var error = function () {...};
ATT.rtc.configure(success, error);
// Configure using the provided options obtained elsewhere
var phone = ATT.rtc.configure({
  ewebrtc_domain: 'my.domain.com'
});
// Configure using the provided options obtained elsewhere
var phone = ATT.rtc.configure({
  ewebrtc_domain: 'my.domain.com',
  api_endpoint: 'https://api.att.com'
});
